使用Hexo+Github搭建个人博客网站

发表于 2023-02-21 更新于 2023-03-02 分类于 博客站务 阅读次数：

前言

大家好，我是梁国庆。

一直有这么一个想法，那就是在互联网上搭建一个属于自己的小空间，用来记录自己的成长经历、学习总结、个人所思等等。

在经过深入了解之后，感觉 Hexo + Github 搭建网站这种方式挺适合本小白的，便开始着手搭建我的个人博客，本篇文章用来记录网站搭建的过程。

以下操作均基于 windows 操作系统。

如果你的电脑中尚未安装所需要的程序，请前往官网下载安装包后，根据安装教程完成安装。

配置 Github

注册账号

首先需要拥有一个 Github 账号，没有的可以注册一个，网上的注册教程挺多的，在这不在赘述。

  • Github 官网

创建仓库

拥有账号后，接下来创建一个仓库，需要注意的点有：

  • 仓库名称需为：xxxx(本人 Gtihub 的用户名).github.io，此处 xxxx 一定要是本人 Gtihub 的用户名。
  • 将仓库设置为公共仓库，即选择：Public。
  • 点击：Create repository，创建即可。

Hexo 安装主题的方式非常简单，只需要将主题文件拷贝至站点目录的 themes 目录下， 然后修改配置文件的 theme 为 next 即可。

关于 Hexo 的更多主题，请前往：https://hexo.io/。

完成以上步骤后，你的网站就可以通过网址访问啦，访问网址为：Github 用户名.github.io。

自定义域名

申请域名

想要自定义域名，首先你要拥有一个域名，没有的可以申请一个，推荐前往：阿里云、腾讯云等进行申请。

解析域名

在你申请域名的网站中，进入控制台找到域名，点击解析域名填入网址为：Github 用户名.github.io 即可。

映射域名

在你的 Github 仓库目录下新建一个 CNAME 文件，文件内容为需要映射的域名，即你上文中申请、解析的域名。

至此，就可以用自己的域名访问博客网站啦。

常见问题

问题 1

问题：如何新建文章？

解决：在 _post 目录下新建 .md 文件，使用 Markdown 语法编写好后，在站点目录重新生成静态文件，部署到 Github 即可。

问题 2

问题：在配置文件里有分类、标签等新加的目录，但是点击网页访问却没有页面。

原因：source 目录下没有相应的文件。

解决：在博客的 source 目录下，新建相应目录。
